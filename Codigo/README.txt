Estructura del código:

- Clustering: El código completo, junto con las instrucciones y un ejemplo de uso.

- Clustering_Experimentacion: El código completo, junto con las instrucciones
			      y los ejemplos expuestos en la memoria. Para recrear
			      algunos ejemplos que no salen, solo habrá que modificar minimanmente
		              algun parametro o los puntos que se crean, pero se verá claramente.


- Objetivos_Especificos: Por cada objetivo especifico, la parte del codigo que cumple esa función.
			 Hay ocasiones en las que se muestra el código antiguo, y en el objetivo 4
			 no se consiguen buenos resultados ya que son necesarias varias iteraciones
			 para ello.

- Clustering_Interfaz_Grafica.ipynb: Si se ejecuta la única celda que se encuentra en este archivo, 
				     aparecerá la interfaz gráfica.

- Clustering_Interfaz_Grafica.py: Si python está configurado como variable de entorno, podrá ejecutarse
			          la interfaz gráfica si así se desea, si no, se podrá lanzar desde la
			 	  consola de jupyter escribiendo su nombre.

- Codigo_Antiguo: Los objetivos específicos y algunas pruebas antes de que se cambiase en gran medida
		  el código, por si se quiere acceder a él (Los nombres de las estructuras y métodos
		  no concordarán).

- Data: Carpeta donde se encuentran los archivos CSV de los puntos.

- Res: Carpeta donde se han ido guardando las imagenes generadas para incluirlas en la memoria.


Tanto en Clustering, Clustering_Experimentación y Clustering_Interfaz_Grafica se encontrarán 
recomendaciones sobre que valores pasar como parámetros, así como en la memoria. También vendrá un
ejemplo de uso, instrucciones, y las pruebas realizadas.


Al ejecutar la interfaz, en el menú Archivo/Instrucciones se podrá encontrar una breve guía
de utilización.
